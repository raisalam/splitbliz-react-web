# Memory MCP Instructions

## On Every Session Start
- Read from memory MCP to recall project context, past decisions, and preferences

## Always Save to Memory
- Architectural decisions made and why
- Approaches rejected and why
- User preferences discovered
- Recurring patterns or shortcuts specific to this project
- Bugs fixed that could recur

## Memory Entry Format
- **Entity:** clear category name (e.g. `splitbliz-frontend`, `splitbliz-backend`)
- **Facts:** specific, actionable statements
- **Date:** when this was recorded
